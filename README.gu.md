# Traffic Redirect — વાસ્તવિક ઉપયોગ કેવી રીતે થાય છે

> English version: [README.md](README.md) · વ્યાપાર વિશ્લેષણ: [docs/STRATEGY.md](docs/STRATEGY.md)

આ પ્રોજેક્ટ **પૈસા કમાવતી ટૂંકી લિંક (monetized short link)** બનાવે છે.

તમે એક લાંબી લિંક આપો → સિસ્ટમ એક ટૂંકી લિંક આપે → કોઈ એ ટૂંકી લિંક ખોલે ત્યારે
પહેલા **થોડી સેકન્ડ માટે એક જાહેરાત વાળું પેજ** દેખાય → પછી એ વ્યક્તિ તમારી ખરી
વેબસાઇટ પર પહોંચે.

એ જાહેરાત જોવાઈ, એટલે તમને Adsterra તરફથી પૈસા મળે.

```
તમે બનાવો:  https://your-app.vercel.app/aB3xK9q  →  https://mysite.com/page
                     ↑ ટૂંકી લિંક                          ↑ ખરી વેબસાઇટ
```

## આમાં બે માણસ છે

આ સિસ્ટમ સમજવા માટે **બે અલગ વ્યક્તિ** ને અલગ સમજવી પડે:

| કોણ | શું કરે | શું જુએ |
|---|---|---|
| **તમે** (admin) | લિંક બનાવો, જાહેરાતનો કોડ નાખો | `/` પર admin પેજ |
| **મુલાકાતી** (visitor) | તમારી ટૂંકી લિંક પર ક્લિક કરે | રાહ જોવાનું પેજ, પછી તમારી સાઇટ |

મુલાકાતીને admin પેજ ક્યારેય દેખાતું નથી. તમારો `ADMIN_TOKEN` જ એ દરવાજાની ચાવી છે.

## ભાગ ૧ — તમે (લિંક બનાવનાર)

### પગલું ૧ — admin પેજ ખોલો

બ્રાઉઝરમાં `https://your-app.vercel.app/` ખોલો. તમને એક કાળું ફોર્મ દેખાશે.

### પગલું ૨ — એક વાર sign in કરો

**Admin token** ના ખાનામાં તમે Vercel માં જે `ADMIN_TOKEN` સેટ કર્યો હતો એ લખો અને
**Sign in** દબાવો.

**હવે પછી ૩૦ દિવસ સુધી આ બ્રાઉઝરમાં ફરી ટોકન લખવો નહીં પડે.** પેજ ખોલશો એટલે સીધું
ખૂલી જશે.

<details>
<summary>આ કઈ રીતે કામ કરે છે? (અને <code>localStorage</code> કેમ નથી વાપર્યું)</summary>

Sign in કરો એટલે સર્વર તમારા બ્રાઉઝરમાં એક **HttpOnly cookie** મૂકે છે. "HttpOnly"
એટલે — **કોઈ પણ JavaScript એ કૂકી વાંચી શકતું નથી.**

આ કેમ જરૂરી? કારણ કે તમારું admin પેજ અને જાહેરાતવાળું પેજ **એક જ origin** પર છે:

```
https://your-app.vercel.app/          ← admin પેજ
https://your-app.vercel.app/aB3xK9q   ← અહીં Adsterra નો script ચાલે છે
```

જો ટોકન `localStorage` માં સાચવ્યો હોત, તો **Adsterra નો script એ વાંચી શકત** — અને
પછી તમારી બધી લિંક બનાવી, જોઈ કે ભૂંસી શકત. એ script તમારો નથી, અને આ ધંધામાં
malware વાળા ad network સામાન્ય છે.

કૂકીમાં તમારો ટોકન નથી — ફક્ત એક સહી કરેલી તારીખ છે. અને એ કૂકી ફક્ત `/api` પર જ
મોકલાય છે, જાહેરાતવાળા પેજ પર ક્યારેય નહીં.

**બહાર નીકળવું હોય તો** ઉપર જમણે **Sign out** દબાવો.
</details>

### પગલું ૩ — ફોર્મ ભરો

| ખાનું | શું નાખવું | જરૂરી? |
|---|---|---|
| **Destination URL** | તમારી ખરી વેબસાઇટ, જ્યાં લોકોને મોકલવા છે | **હા** |
| **Adsterra ad script** | Adsterra માંથી કૉપી કરેલો `<script>` કોડ (Social Bar / Banner / Popunder) | બેમાંથી ઓછામાં ઓછું એક |
| **Adsterra Direct Link** | Adsterra ની Direct Link URL | બેમાંથી ઓછામાં ઓછું એક |
| **Page title** | રાહ જોવાના પેજ પર ઉપર દેખાતું લખાણ | ના (default: "Preparing your link") |
| **Wait (sec)** | કેટલી સેકન્ડ રાહ જોવડાવવી (૦ થી ૩૦) | ના (default: ૫) |

**ad script અને Direct Link — બંનેમાંથી ઓછામાં ઓછું એક ભરવું જ પડશે.** બંને ખાલી હોય
તો સિસ્ટમ લિંક બનાવવાની ના પાડી દેશે. કારણ સીધું છે: જાહેરાત વગરની લિંક કમાણી નથી કરતી,
તો પછી વચ્ચે રાહ જોવડાવવાનો કોઈ અર્થ નથી.

### પગલું ૪ — "Create link" દબાવો

તમને એક ટૂંકી લિંક મળશે, જેમ કે `https://your-app.vercel.app/aB3xK9q`.
`aB3xK9q` એ **રેન્ડમ ૭ અક્ષર** છે — કોઈ અંદાજ લગાવી ન શકે.

આ લિંક હવે તમે ગમે ત્યાં મૂકો: YouTube description, Telegram, WhatsApp, બ્લોગ, જ્યાં પણ.

### પગલું ૫ — આંકડા જુઓ

નીચે **"Your links"** માં **Refresh** દબાવો. દરેક લિંક નીચે ચાર આંકડા દેખાશે:

```
42 clicks · 30 new · 22 ad impressions · 6 direct opens
```

આનો અર્થ [નીચે સમજાવ્યો છે](#ગણતરીના-આંકડા-counters-નો-અર્થ).

## ભાગ ૨ — મુલાકાતી (લિંક પર ક્લિક કરનાર)

હવે કોઈ વ્યક્તિ તમારી લિંક પર ક્લિક કરે ત્યારે શું થાય? **બે અલગ રસ્તા છે.**

### રસ્તો A — પહેલી વાર આવેલો માણસ

```
૧. એ ક્લિક કરે  →  https://your-app.vercel.app/aB3xK9q

૨. એને "રાહ જુઓ" વાળું પેજ દેખાય:
   ┌──────────────────────────────┐
   │   Preparing your link        │
   │   ▓▓▓▓▓▓░░░░░░░░░░░░  (bar)  │
   │                              │
   │   [ તમારી Adsterra જાહેરાત ]  │  ← અહીં પૈસા બને છે
   │                              │
   │   [ Please wait… 3s ]        │  ← બટન બંધ છે
   │   Destination: mysite.com    │
   └──────────────────────────────┘

૩. ૫ સેકન્ડ પૂરી થાય → બટન ચાલુ થાય → "Continue →"

૪. એ Continue દબાવે
   ├─ Direct Link હોય તો નવી ટૅબમાં ખૂલે  (એના પોતાના ક્લિકથી જ, આપોઆપ નહીં)
   └─ અને એ તમારી ખરી વેબસાઇટ પર પહોંચી જાય
```

### રસ્તો B — પહેલા આવી ચૂકેલો માણસ

```
એ ક્લિક કરે  →  કોઈ રાહ નહીં, કોઈ જાહેરાત નહીં  →  સીધો તમારી વેબસાઇટ પર
```

**૭ દિવસ સુધી** એને ફરી જાહેરાત નહીં દેખાય. એના બ્રાઉઝરમાં `seen_aB3xK9q` નામની એક
નાની કૂકી મુકાઈ જાય છે, અને એ કૂકી જોઈને સિસ્ટમ એને સીધો જવા દે છે.

> ⚠️ **અહીં એક મોટી વાત છે.** રસ્તો B માં તમને **એક પણ રૂપિયો મળતો નથી** — કારણ કે
> જાહેરાત જ દેખાતી નથી. એટલે "user retention" (લોકો પાછા આવે) અને "કમાણી" —
> આ બે વસ્તુ અહીં એકબીજાની વિરુદ્ધ છે. વધુ માટે [ચેતવણીઓ](#ખૂબ-અગત્યની-ચેતવણીઓ) જુઓ.

## પડદા પાછળ શું થાય છે

### ટોકન — જે ગેમ સાચી રાખે છે

આ સૌથી અગત્યની વાત છે, અને દેખાતી નથી.

જ્યારે મુલાકાતીને રાહ જોવાનું પેજ મોકલાય, ત્યારે સર્વર એની સાથે એક **સહી કરેલો
ગુપ્ત ટોકન** (HMAC signed token) પણ મોકલે છે. એ ટોકનમાં લખેલું હોય છે કે "આ પેજ
બરાબર **આ ક્ષણે** આપવામાં આવ્યું હતું."

પછી જ્યારે એ Continue દબાવે અને `/aB3xK9q/go` પર જાય, ત્યારે સર્વર બે વસ્તુ તપાસે:

1. **આ ટોકન સાચો છે?** (કોઈએ બનાવટી તો નથી બનાવ્યો ને?)
2. **આ ટોકન ઓછામાં ઓછો ૫ સેકન્ડ જૂનો છે?** (એટલે કે માણસે ખરેખર રાહ જોઈ?)

બેમાંથી એક પણ ખોટું હોય → એને પાછો રાહ જોવાના પેજ પર ધકેલી દેવાય.

**આ કેમ જરૂરી છે?** પહેલા આ તપાસ **નહોતી**. એટલે કોઈ પણ માણસ સીધું બ્રાઉઝરમાં
`your-app.vercel.app/aB3xK9q/go` ટાઇપ કરીને **જાહેરાત જોયા વગર** તમારી સાઇટ પર
પહોંચી જતો. ફક્ત ત્રણ અક્ષર `/go` લખીને. તમારી આખી કમાણી છટકી જતી.

એથી પણ ખરાબ: `/aB3xK9q/adview` પર કોઈ પણ વારંવાર request મોકલીને તમારો
"ad views" નો આંકડો ગમે તેટલો વધારી શકતું. પછી તમે Adsterra ને એવી જાહેરાતોના
આંકડા આપો જે ખરેખર કોઈએ જોઈ જ નહોતી — **એને impression fraud કહેવાય, અને એ જ
કારણથી આવા account બંધ થાય છે.**

હવે બંને દરવાજા બંધ છે.

### Bot ઓળખ

Google નું crawler, WhatsApp/Telegram/Discord નું "link preview" — આ બધા તમારી લિંક
આપોઆપ ખોલે છે. એ માણસ નથી.

એટલે સિસ્ટમ એમને પેજ તો બતાવે છે (જેથી preview બરાબર દેખાય), પણ **એમની ગણતરી નથી
કરતી**. નહીંતર તમારો clicks નો આંકડો ખોટો હોત, અને તમે Adsterra ને એવી જાહેરાતોના
પૈસા માંગત જે કોઈ માણસે જોઈ જ નથી.

### દરેક ક્લિકનો રેકોર્ડ

હવે દરેક (અસલી) ક્લિક પર આટલી માહિતી સેવ થાય છે:

`slug, સમય, દેશ, રાજ્ય, શહેર, referrer, browser, પાછો આવેલો કે નહીં`

**દેશ (country) સૌથી કીમતી માહિતી છે.** કારણ નીચે.

## પૈસા ક્યાંથી આવે છે

તમને પૈસા **Adsterra** આપે છે, આ સિસ્ટમ નહીં. આ સિસ્ટમ ફક્ત જાહેરાત બતાવવાની જગ્યા
બનાવે છે.

બે રીતે:

| રીત | ક્યાં દેખાય | ક્યારે ગણાય |
|---|---|---|
| **Ad script** | રાહ જોવાના પેજ પર જ | પેજ ખૂલ્યાના ૨.૫ સેકન્ડ પછી (જાહેરાત load થવાનો સમય) |
| **Direct Link** | Continue દબાવો ત્યારે નવી ટૅબમાં | `/go` પર પહોંચે ત્યારે |

> Direct Link **હંમેશા મુલાકાતીના પોતાના ક્લિકથી** ખૂલે છે. આપોઆપ ક્યારેય નહીં,
> iframe માં ક્યારેય નહીં. એ બંને કરવું એટલે છેતરપિંડી, અને Adsterra તરત account
> બંધ કરી દે.

### કમાણીનું ખરું ગણિત

```
કમાણી = (જાહેરાત જોનારા લોકો ÷ 1000) × eCPM
```

`eCPM` એટલે ૧૦૦૦ વાર જાહેરાત દેખાય તેના કેટલા ડૉલર. **અને એ દેશ પ્રમાણે ધરખમ બદલાય છે:**

| દેશ | આશરે દર (per 1000 views) |
|---|---|
| 🇺🇸 અમેરિકા | ~$18 |
| 🇬🇧 બ્રિટન | ~$15 |
| 🇮🇳 ભારત | ~$4 |
| બાકીના મોટા ભાગના દેશ | ~$1.50 – $4.50 |

**એટલે અમેરિકાનો એક મુલાકાતી = ભારતના ~૪ મુલાકાતી જેટલી કમાણી.**

પેજની ડિઝાઇન, countdown કેટલી સેકન્ડનું છે, જાહેરાત ક્યાં મૂકી — આ બધું ભેગું મળીને
લગભગ ૩% નો ફરક પાડે છે. **દેશ ૧૮ ગણો ફરક પાડે છે.**

એટલા માટે જ હવે દરેક ક્લિકનો દેશ સેવ થાય છે.

## ગણતરીના આંકડા (counters) નો અર્થ

Admin પેજ પર દેખાતા ચાર આંકડા:

| આંકડો | અર્થ |
|---|---|
| **clicks** | કુલ કેટલા (અસલી) માણસોએ લિંક ખોલી. પાછા આવેલા પણ ગણાય. Bot ન ગણાય. |
| **new** | કેટલા **પહેલી વાર** આવ્યા. **આટલા લોકોએ જ જાહેરાત જોઈ.** |
| **ad impressions** | script વાળી જાહેરાત ખરેખર કેટલી વાર દેખાઈ (ટોકનથી ચકાસેલી) |
| **direct opens** | Direct Link કેટલી વાર નવી ટૅબમાં ખૂલી |

**સૌથી અગત્યનો આંકડો `new` છે, `clicks` નહીં.** કારણ કે કમાણી ફક્ત નવા મુલાકાતીથી થાય છે.

`clicks` ૧૦૦૦ હોય પણ `new` ફક્ત ૫૦ હોય, તો તમે ૫૦ વારની જ કમાણી કરી.

> **નોંધ:** પહેલા `ad impressions` અને `direct opens` એક જ આંકડામાં ભેગા હતા. એટલે
> જે લિંકમાં બંને પ્રકારની જાહેરાત હોય, ત્યાં **એક જ માણસ બે વાર ગણાઈ જતો**. હવે
> અલગ છે — જેથી તમે Adsterra ના પોતાના dashboard સાથે સરખાવી શકો.

## પહેલી વાર સેટઅપ

### ૧. Redis બનાવો (Railway પર)

1. [railway.app](https://railway.app) → **New Project → Database → Add Redis**
2. Redis service ખોલો → **Variables** tab
3. **`REDIS_PUBLIC_URL`** ની કિંમત કૉપી કરો

> ⚠️ **`REDIS_URL` નહીં — `REDIS_PUBLIC_URL` કૉપી કરવો.**
> Railway નો `REDIS_URL` ફક્ત Railway ના પોતાના network ની અંદરથી જ પહોંચે છે.
> તમારી app Vercel પર ચાલે છે, જે બહાર છે. ખોટો URL નાખશો તો app અટકી જશે અને
> timeout થશે — અને error message માંથી કારણ નહીં સમજાય.

### ૨. Vercel પર deploy કરો

1. આ repo GitHub પર push કરો
2. [vercel.com](https://vercel.com) → **Add New → Project** → repo import કરો
3. **Environment Variables** માં ઉમેરો:

| નામ | જરૂરી? | શું નાખવું |
|---|---|---|
| `ADMIN_TOKEN` | **હા** | તમે નક્કી કરેલો લાંબો, અઘરો પાસવર્ડ |
| `REDIS_URL` | **હા** | Railway માંથી કૉપી કરેલો **PUBLIC** URL |
| `GO_SECRET` | ના | ટોકનની સહી માટેની ચાવી. ન નાખો તો `ADMIN_TOKEN` વપરાય. |
| `SEEN_DAYS` | ના | કેટલા દિવસ જાહેરાત ન બતાવવી. Default `7`. **આ સીધો કમાણીનો knob છે.** |
| `SESSION_DAYS` | ના | કેટલા દિવસ sign in રહેવું. Default `30`. |

4. **Deploy** દબાવો

### ૩. પહેલી લિંક બનાવો

`https://<your-app>.vercel.app/` ખોલો, એક વાર ટોકન નાખીને Sign in કરો, બની ગયું.

### ૪. પોતાની જાતે તપાસો

```bash
npm install
npm test        # ૫૩ ટેસ્ટ ચાલશે, Redis કે internet ની જરૂર નથી
```

## ખૂબ અગત્યની ચેતવણીઓ

### ૧. પાછો આવેલો માણસ = શૂન્ય કમાણી

`SEEN_DAYS=7` નો સીધો અર્થ: એક વાર જે માણસે જાહેરાત જોઈ લીધી, એ **આખા અઠવાડિયા સુધી**
તમને એક રૂપિયો પણ નહીં આપે.

- આંકડો **ઘટાડો** (દા.ત. `SEEN_DAYS=1`) → વધુ કમાણી, પણ લોકોને વધુ ચીડ ચડે
- આંકડો **વધારો** → લોકો ખુશ, પણ કમાણી ઓછી

**આ બે વસ્તુ સાથે નથી મળતી.** કોઈ એવી setting નથી જે બંને જીતે. તમારે નક્કી કરવાનું છે.

### ૨. "unique visitors" નો આંકડો સાચો નથી

એ ફક્ત કૂકી પર આધારિત છે. કોઈ incognito માં ખોલે, કૂકી delete કરે, બીજું બ્રાઉઝર
વાપરે — બધામાં એ ફરી "નવો" ગણાઈ જાય.

### ૩. Bot ઓળખ નબળી છે

અત્યારે ફક્ત browser નું નામ (user-agent) જોઈને bot ઓળખાય છે. જે માણસ જાણી જોઈને
પોતાનું બ્રાઉઝર નામ બદલી નાખે, એ પકડાશે નહીં.

ખરી fraud detection (IP ની શાખ, ASN, ક્લિકનો સમય-pattern) અહીં **નથી**. અને ad
network આ જ કારણથી account બંધ કરે છે.

### ૪. જાહેરાતનો કોડ સીધો પેજમાં નખાય છે

`adScript` નું ખાનું જે લખો એ જેમનું તેમ પેજમાં જાય છે — કારણ કે જાહેરાતનો કોડ એવો
જ હોય. **એટલે ફક્ત ભરોસાપાત્ર જગ્યાએથી જ કોડ કૉપી કરો.** જેની પાસે `ADMIN_TOKEN` છે
એ તમારા પેજ પર કોઈ પણ JavaScript ચલાવી શકે.

### ૫. Ad-blocker

દુનિયાના લગભગ **૩૦% લોકો** ad-blocker વાપરે છે (ભારત/દક્ષિણ-પૂર્વ એશિયામાં તો ૬૫% થી
વધુ). એમને જાહેરાત દેખાતી જ નથી, એટલે એમનાથી કમાણી થતી નથી.

વધુ ઊંડી ચર્ચા માટે [docs/STRATEGY.md](docs/STRATEGY.md) વાંચો — એમાં આ ધંધાની ખરી
હકીકત, હરીફો, અને એક વધુ સારો રસ્તો સમજાવ્યો છે.

## વારંવાર પુછાતા પ્રશ્નો

**પ્ર: લિંક બનાવ્યા પછી destination બદલી શકાય?**
ના. અત્યારે એ સુવિધા નથી. લિંક delete કરીને નવી બનાવવી પડે (નવો slug મળશે).

**પ્ર: મારી લિંક `/go` લગાવીને કોઈ છટકી જાય તો?**
હવે નહીં. સહી કરેલા ટોકન વગર `/go` કામ નથી કરતું — એ માણસને પાછો રાહ જોવાના પેજ પર
મોકલી દેવાય છે.

**પ્ર: Redis બંધ થઈ જાય તો બધી લિંક બગડી જાય?**
ના. સિસ્ટમ થોડી વાર માટે જૂની (cached) માહિતી યાદ રાખે છે, અને Redis ન મળે તો પણ
મુલાકાતીને destination પર મોકલી દે છે. જૂની માહિતીથી redirect કરવું એ error પેજ
બતાવવા કરતાં સારું છે.

**પ્ર: `clicks` અને `new` વચ્ચે આટલો ફરક કેમ?**
`clicks` માં પાછા આવેલા લોકો પણ ગણાય છે, `new` માં ફક્ત પહેલી વારના. અને કમાણી ફક્ત
`new` થી થાય છે.

**પ્ર: Adsterra નું dashboard અને આ આંકડા સરખા કેમ નથી?**
ક્યારેય સંપૂર્ણ સરખા નહીં થાય. Adsterra પોતાની રીતે invalid traffic કાઢી નાખે છે.
પણ હવે `ad impressions` અને `direct opens` અલગ હોવાથી તમે ઓછામાં ઓછું **સરખાવી તો
શકો છો** — પહેલા એ પણ શક્ય નહોતું.

**પ્ર: WhatsApp માં લિંક મોકલું તો preview કેમ દેખાય છે?**
WhatsApp નું bot તમારી લિંક ખોલીને title વાંચે છે. એ ગણતરીમાં નથી આવતું — clicks
નથી વધતા, જાહેરાત નથી ગણાતી.

**પ્ર: વધુ કમાણી માટે સૌથી પહેલું શું કરવું?**
તમારા મુલાકાતી **કયા દેશના છે** એ જુઓ. હવે એ માહિતી સેવ થાય છે. જો મોટા ભાગના
tier-3 દેશના હોય, તો પેજમાં ગમે તેટલા ફેરફાર કરો — કમાણી નહીં વધે. **ટ્રાફિક ક્યાંથી
આવે છે એ બદલવું પડે, પેજ નહીં.**
